# Glenda

Glenda is a program that plays [Flood
Wars](https://play.google.com/store/apps/details?id=com.fva.flood.wars) with
other programs. This is my entry for a [Flood Wars
contest](http://algopedia.ro/wiki/index.php/Concurs_Flood_Wars_2015,_clasele_7/8).

## Current stage

This program is very, very WIP. For now, it moves randomly.

## The game

The game is played between two players on a board. The first player starts in
the bottom-left corner, and the second in the top-right. Each square on the
board has a color (`@ # + . *`). When a player changes his square's color, all
squares in the neighborhood change their colors (like in a flood fill). The game
ends when there are only two colors on the board, one for each player. The
player who has the more squares with the same color wins.

## The program

* The board's size is at least 10x10 and at most 50x50 squares.
* Each player starts with a different color.
* The players can select from only three colors out of five at their turn. They
can't select their own color or the opponent's.
* The program's input won't contain any whitespace or characters except for the
board and the end of line character (`\n`) at the end of every line
(including the last one).

## Example boards

```
.#+#.*.@.@
@.+*@.+*#+
**#++@**#@
#@#@.@@+@#
++@++@#.@.
**##*@*#++
*@@*.+*.*+
```

In this example, the bottom-left player selected the `*` color. He couldn't
select `@` because that's the color of the opponent.

```
.#+#.*.@.+
@.+*@.+*#+
**#++@**#@
#@#@.@@+@#
++@++@#.@.
**##*@*#++
*@@*.+*.*+
```

Now, the top-right player selected the `+` color and "absorbed" the `+`
below.

```
.#+#.*.@.+
@.+*@.+*#+
**#++@**#@
#@#@.@@+@#
++@++@#.@.
####*@*#++
#@@*.+*.*+
```

The bottom-left player selected `#` and absorbed two more squares.

The game ends when all the squares were absorbed by the players, or after 150
moves.

## Who am I? Is it my turn to move?

To be able to distinguish between players, the first line of the input
contains a char for the player's position. `J` is bottom-left ("jos" in
Romanian means "down"). `S` is top-right ("sus" in Romanian means "up").

When outputting the current board to the standard output, the program must
output the character for the other player (`J` if the current player is `S`,
for example).

The program accepts the following input:

```
J
.#+#.*.@.@
@.+*@.+*#+
**#++@**#@
#@#@.@@+@#
++@++@#.@.
**##*@*#++
#@@*.+*.*+
```

And outputs:

```
S
.#+#.*.@.@
@.+*@.+*#+
**#++@**#@
#@#@.@@+@#
++@++@#.@.
**##*@*#++
*@@*.+*.*+
```

## Scoring

1. The game ends in the following situations:

  1. There are only two colors on the board, one for each player.

  2. 150 moves were made, 75 for each player.

2. The score is based on the number of the player's squares plus the number of
squares that can be absorbed only by the player. The score is number of
player's squares - number of opponents squares.

    Example:

    ```
    J
    ..........
    .@.....*..
    .+@@+..#..
    @@.@+.....
    @@@@@@@@@@
    @@*@@@##@@
    @@@@@@@@@@
    ```

    Here, the score of `J` is the number of squares of his color, 32 squares
    of `@` + squares that can only be absorbed by him: 1 `*` + 1 `.` + 2 `#`
    = 36.

    You can see one `@` was ignored because it wasn't absorbed by the
    player. Squares can be absorbed only vertically or horizontally.

    The score of `S` is 28 `.` + 1 `*` + 1 `#` = 30.

    In this example, `J` wins by 6 points (6 = 36 - 30).

3. If one's program makes a bad move or takes too much to make a move, the
   opponent receives `width * height` points. If the board is 50x50 and
   player 1 fails, player 2 receives 50 * 50 = 2500 points.

## Contest rules

1. Programs must be written in C. No C++!
2. Each move must be made in a second.
3. Memory limit is 128MB.
4. The program reads from the standard input (`stdin`) and writes to the
   standard output (`stdout`). The program isn't allowed to create files.
5. The program isn't allowed to read or write to/from files.
6. Input data is always correct. It won't contain any whitespace or
   characters except for the board and the end of line character (`\n`) at the
   end of each line.
7. The input ends in `EOF` (End Of File).
